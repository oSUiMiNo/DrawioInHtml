LiveApp（URP8）

Unity 6 / URP 上で動く VTuber 配信向け統合アプリ。アバター・ワールド・家具などのコンテンツを「パック」単位で外部から取り込み、シーンに合成しながら配信する。

Unity 6 / URP 17.2+ UniRx UniTask UI Toolkit Addressables DOTween VMC Protocol

UIOverlaying をオン／オフする責務

タイミング誰が立てる誰が反応する
UI Toolkit パネル OnEnable PackPanelViewBase が UIOverlaying=true UIRayCasterManager：Camera.main.UIRayCaster.enabled=false＋ Blur 表示
RuntimeInputP：ゲートが入力遮断

その直後に UIRayCaster.enabled=false を自前で再上書き		 Blur 即消滅。UIRayCaster だけはマウスアップ＋0.3 秒遅延で復帰
注意：UIRayCaster は内部に 0.25 秒のダブルクリック判定窓を持つため、1 フレーム遅延だと「閉じた瞬間の同じクリックが背後 3D に届いて UI が即再オープン」する。マウスアップ＋0.3 秒で復帰しないと貫通する（実装の注意点と回避策.md §3-7）。

ゲート実装の中身

// Presenter/RuntimeInputP.cs
static System.Action G(System.Action a) => () => {
    if (UIOverlaying.Value) return;
    a();
};

// 使い方：全 InputEventHandler 購読を G(...) で包む
InputEventHandler.On_F  += G(() => onVRM_Move.OnNext(Vector3.down));
InputEventHandler.On_Wheel += G(() => onCAM_Distance.OnNext(Unit.Default));
InputEventHandler.OnDown_H += G(() => onCAM_Home.OnNext(Unit.Default));
新しい入力ハンドラを RuntimeInputP に追加するときは必ず G(...) で包む。包み忘れると UI 表示中もその入力だけ素通りしてバグの原因になる。
