Drawio in HTML — 開発者向けドキュメント

本ファイルは「拡張機能を改修したい・コントリビュートしたい・内部の動きを理解したい」開発者向けの技術ドキュメント。 使い方を知りたい一般ユーザーは README.ja.md を参照してください。

本ファイル自体が拡張機能のショーケース 下のアーキテクチャ図は <script type="application/xml" id="..."> として本HTMLに直接埋め込まれている。
  • ブラウザで開いた場合：CDN viewer + 自前マウントで描画される
  • 本拡張機能で開いた場合：拡張描画に切り替わり、ホバーで 🔍/✏️ アイコンが現れる

技術的位置づけ

Drawio in HTML は、VSCode の Custom Text Editor + Webview Panel 二段構えで Drawio 埋め込み HTML を扱う拡張機能。

  • プレビュー側：CustomTextEditorProvider を使い、ユーザの HTML 本文をそのまま WebView に表示しつつ、HTML 内の Drawio マーカーを <div class="drawio-slot"> に置換して同梱 viewer (viewer-static.min.js) で SVG レンダリング。
  • 編集側：「✏️」を押すと WebviewPanel が ViewColumn.Beside に開き、その中の iframe で embed.diagrams.net（公式 Drawio オンラインエディタ）をロード。postMessage で init → load → save → exit をブリッジ。
  • 保存：Drawio 側の save を受信したら、拡張ホストが htmlPatcher で該当 <script> の中身だけを書き換え、WorkspaceEdit + doc.save() でディスクへ自動永続化。

Drawio エディタ機能を持つ既存 VSCode 拡張（hediet.vscode-drawio 等）には依存していない。本拡張機能は完全に独立。

対応マーカー仕様（v0.5+）

v0.5 で拡張専用マーカーは廃止され、認識する形式は1種類のみに統一された。 

<script type="application/xml" id="X">
<mxfile>...</mxfile>  <!-- または <mxGraphModel>...</mxGraphModel> -->
</script>
  • HTML5 標準の「インラインデータブロック」用法。ブラウザは type="application/xml" を未知扱いで実行しないので、副作用ゼロで HTML に置ける
  • 編集識別子は id 属性
  • 中身が <mxfile> または <mxGraphModel> で始まる場合のみ Drawio として扱う（汎用 XML データを誤認しないため）
v0.4 以前との非互換 data-drawio-id（v0.3+ で導入した拡張専用属性）や type="application/drawio+xml"（v0.2.x の旧形式）は認識されません。これらを使った既存 HTML は id 属性形式に書き換えてください。XML 中身（<mxfile>...）は変更不要。

ブラウザでも描画する自前マウント

ブラウザで開いた時にも図を表示したい場合は、CDN viewer と自前マウント JS を組み合わせます。本ドキュメント下部のアーキテクチャ図がこの構成の生きた例です。 

<div class="mxgraph" id="diagram-host-architecture"></div>

<script type="application/xml" id="architecture">
<mxfile>...</mxfile>
</script>

<!-- viewer-static.min.js は data-mxgraph 属性経由で設定を受け取る。
     viewer ロード前に同期で属性を付与しておけば、
     DOMContentLoaded 時の auto processElements が拾って描画してくれる。 -->
<script>
  (function() {
    var xml = document.getElementById('architecture').textContent.trim();
    document.getElementById('diagram-host-architecture').setAttribute(
      'data-mxgraph',
      JSON.stringify({ xml: xml, toolbar: null, 'auto-fit': true, center: true })
    );
  })();
</script>

<script src="https://viewer.diagrams.net/js/viewer-static.min.js"></script>

本拡張機能で開いた時の挙動：

  • id="architecture" の script を編集対象として認識し、拡張描画スロットを作る
  • ユーザ自前の <div class="mxgraph"> ホストは CSS .mxgraph:not(.drawio-rendered) { display: none } で自動非表示
  • ユーザ自前の <script src=".../viewer-static.min.js"> は previewHtmlBuilder が DOM から削除（二重ロード防止）
  • 結果として、拡張描画スロットだけが表示される

アーキテクチャ概略

下の図は本ドキュメント自身の「自前マウント」例です。ブラウザでも本拡張機能でも描画されます。

[VSCode] ├─ プレビュータブ（CustomTextEditor） ← HTML本文+Drawioインライン表示 │ └─ Webview "preview" │ ├─ ユーザのHTML本文をベースにレンダリング（<head>/<body> 保持） │ ├─ <meta CSP>、preview.css、viewer-static.min.js、preview.js を注入 │ ├─ Drawio script タグを <div class="drawio-slot"> に置換 │ ├─ 相対パスの <img>、<link>、<a> 等を webview.asWebviewUri() で自動変換 │ └─ slot 内に viewer SVG を生成 / ホバー時オーバーレイ 🔍/✏️ │ └─ クリック → 拡張本体へ postMessage │ └─ 編集タブ（WebviewPanel, ViewColumn.Beside） ← 1図1タブ └─ Webview "editor" └─ embed.diagrams.net iframe で Drawio 公式エディタ └─ 保存時に postMessage で XML を拡張本体へ送る └─ 拡張本体 (htmlPatcher) が <script> の中身を書き換え └─ doc.save() でディスクへ自動保存

ソースからの開発

git clone https://github.com/oSUiMiNo/DrawioInHtml.git
cd DrawioInHtml
npm install            # postinstall で viewer-static.min.js (~3.6 MB) が自動取得される
npm run compile
# VSCode でフォルダを開いて F5（Extension Development Host が起動）

動作確認手順：

  1. F5 で Extension Development Host が立ち上がる
  2. 適当な HTML ファイル（<script type="application/xml" id="..."> + <mxfile> を含む）を開き、右クリック → Open With → Drawio HTML Editor
  3. 図がインライン表示され、ホバーで 🔍/✏️ が出ることを確認
  4. ✏️ を押すと右側に編集タブが開き、embed.diagrams.net がロードされる
  5. 編集して保存 → 元の HTML が自動更新される

viewer-static.min.js の取り扱い

本拡張機能は Drawio 公式の viewer-static.min.js をリポジトリに同梱しており、ネット非依存でプレビュー表示できる。 編集側は embed.diagrams.net をオンラインで使うため、編集時のみネット必須。

同梱のしくみ

  • scripts/fetch-viewer.js が https://viewer.diagrams.net/js/viewer-static.min.js からダウンロードして media/viewer-static.min.js に保存
  • package.json の postinstall フックで npm install 時に自動実行
  • 既存ファイルがあり 100KB 以上なら冪等にスキップ
  • 取得失敗時は exit 1。CIで気づけるようにしている

Drawio 本家がアップデートされたら？

機能更新方式
編集（embed.diagrams.net） jgraph がサーバ側を更新するので自動で最新版になる。ただし postMessage プロトコル（init/load/save/exit）が変わると media/editor.js の橋渡しが壊れる可能性あり（プロトコルは長年安定）
プレビュー描画（viewer-static.min.js） 同梱なので自動では更新されない。npm run fetch-viewer で再取得（既存ファイルがあるとスキップされるため、一旦削除してから実行）→ 再ビルド → vsce publish

CSP 仕様（previewHtmlBuilder の挙動）

ユーザHTMLの既存 <meta http-equiv="Content-Security-Policy"> は削除し、拡張機能側で以下の CSP を強制注入： 

default-src 'none';
img-src ${webview.cspSource} https: data: blob: vscode-webview-resource:;
style-src ${webview.cspSource} 'unsafe-inline' https:;
font-src ${webview.cspSource} data: blob: https:;
script-src ${webview.cspSource} 'unsafe-eval' 'unsafe-inline' https:;
connect-src ${webview.cspSource} blob: data: https:;
worker-src ${webview.cspSource} blob: https:;
child-src blob: https:;
frame-src https:;
v0.2.3 以降のセキュリティトレードオフ 'unsafe-inline' と https: を許可しているため、ユーザHTML内のインラインスクリプトや CDN スクリプトが そのまま実行される。Mermaid 等の CDN ライブラリを動かすための仕様。 nonce は使っていない（CSP3 仕様で 'unsafe-inline' と nonce が共存すると nonce が優先され、ユーザのインラインスクリプトが動かなくなる）。 この仕様は README.ja.md のセキュリティ注意 でエンドユーザに明示している。

保存往復の postMessage プロトコル

方向イベント意味
iframe → editor.jsinitDrawio エディタ起動完了。拡張側から XML を load する合図
editor.js → iframeload編集対象 XML を渡す（autosave: 0）
iframe → editor.jssave保存ボタン押下時、新しい XML を含む
editor.js → 拡張ホストsave同 XML を EditorPanelManager へ転送
拡張ホスト—htmlPatcher.replaceDrawioXml で該当 <script> を書き換え、WorkspaceEdit + doc.save()
iframe → editor.jsexit編集タブを閉じる合図

外部編集（ユーザが HTML を別エディタで書き換えた等）への追従は EditorPanelManager.onDocumentChanged が担当。 自分の save で発火した変更を区別するため、entry.suppressNextChange フラグで一度だけ無視する。

テーマ追従の仕組み

  • preview.css を CSP meta の直後（=ユーザ <style> よりも前）に注入。同じ specificity (0,0,1) ならユーザ指定が後勝ちで尊重される設計
  • ユーザが未指定のときだけ body { background: var(--vscode-editor-background); color: var(--vscode-editor-foreground); } が効く
  • <meta name="color-scheme" content="light dark"> をユーザ未指定時のみ自動付与（スクロールバー・フォームコントロールもテーマ追従）
  • Drawio 描画自体も prefers-color-scheme: dark を検出して dark-mode: true を viewer に渡す（v0.3.6 以降）。SVG 背景もダーク化される

トラブルシューティング（開発側）

Output channel "Drawio HTML"

拡張機能の WebView 側 (preview.js / editor.js) からのログは postMessage('log', level, message) 経由で拡張ホストに送られ、vscode.window.createOutputChannel('Drawio HTML') に出力される。 描画エラー、GraphViewer 未定義、SVG 生成失敗などはここに記録される。level: 'error' なら自動でパネルが開く。

拡張機能を更新したのに新バージョンで動かない

  • 症状：拡張機能タブのバージョン表示は最新（例 v0.5.0）なのに、プレビューでは旧版の挙動が続いている
  • 原因：プレビュータブが開いた状態だと VSCode の Extension Host を停止できず、古いプロセスが動き続ける。「Extension host was not stopped because of veto」が VSCode のログに出ているのがサイン
  • 対策：Ctrl+Shift+P → Developer: Reload Window でウィンドウごとリロード。プレビュータブを閉じてから更新するのが確実

npm install で viewer 取得が失敗する

  • postinstall でネット切断・プロキシなどにより失敗するとインストール自体は完了するが media/viewer-static.min.js が存在しない
  • 確認：ls -la media/viewer-static.min.js でファイル存在＆サイズ（実測 3.8MB 程度）を見る
  • 再取得：手動で node scripts/fetch-viewer.js 実行、または上記 URL からダウンロードして所定位置に配置

viewer の二重ロード

  • 症状：図が極端に小さい、編集ボタンが出ない、描画位置がずれる
  • 原因：ユーザHTMLが CDN から viewer-static.min.js をロードしており、拡張同梱版と競合
  • 対策：previewHtmlBuilder が viewer-static.min.js を src に含む <script> を DOM から削除する（v0.3.3 以降）。ソースHTMLは無変更

編集タブが blank

  • ネット切断・社内プロキシで embed.diagrams.net がブロックされている可能性
  • ブラウザの DevTools（VSCode 上では F1 → "Open Webview Developer Tools"）で iframe のエラーを確認

HTML タブが dirty マークになる

  • 通常は EditorPanelManager.applySave 内の doc.save() が走るので dirty にならない
  • 稀に失敗するとユーザに警告ダイアログが出る。手動で Ctrl+S すれば確実に永続化

拡張機能の公開フロー

Marketplace へ publish

# 1. package.json の version を上げる（必須）
# 2. CHANGELOG.md に新バージョン項目を追加
# 3. 必要なら README.md など同梱対象ファイルを更新
npm run compile
npx vsce publish
README は .vsix 同梱版が表示される Marketplace ページの README は GitHub リポジトリを直接見ているのではなく、vsce publish 時に .vsix に同梱された README.md が表示される。GitHub に push しただけでは Marketplace の表示は更新されない。README の修正を Marketplace に反映するには新バージョンで再 publish する必要がある。

同梱対象（.vscodeignore で除外していないもの）

  • out/*.js：TypeScript ビルド成果物
  • media/*：viewer 含む WebView 資産
  • README.md / CHANGELOG.md / LICENSE / package.json
  • 必要な node_modules/（vsce が npm list --production で判定）

同梱しない（.vscodeignore で除外）

  • src/*.ts、tsconfig.json：TypeScript ソース
  • README.ja.md / README.dev.html：開発者向け／日本語版（.vscodeignore で除外）
  • sample/、Test.html、MyDocs/、.claude/：開発時資産
