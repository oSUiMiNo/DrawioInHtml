ポータブルなサンプル（ブラウザでも見える書き方）

このHTMLは ブラウザで直接開いても図が表示される ように書かれています。 VSCode の Drawio in HTML 拡張で開いた場合は、拡張側のリッチ描画（リサイズ追随・拡大・編集）に切り替わります。

仕組み： ブラウザ用の描画コード（公式 viewer-static.min.js + <div class="mxgraph">）と、 拡張が編集対象を識別するためのマーカー（<script type="application/xml" data-drawio-id="...">）を 両方書きます。拡張で開いた時は、自前 mxgraph div が自動で隠され、拡張のリッチ描画が動きます。

図1: シンプルなノード

使い方

  1. ブラウザでこのHTMLをダブルクリックすると、jsdelivr/diagrams.net CDN から viewer が読まれて図が見える
  2. VSCode（Drawio in HTML 拡張）で開くと、自前 mxgraph div が自動で隠され、拡張側のリッチ描画＋編集ボタンが出る
  3. 編集して保存すると、<script type="application/xml" data-drawio-id="..."> の中身が書き換わる

※ 自前 mxgraph div の data-mxgraph 属性内の XML と、マーカーの XML は同じ内容を二重に書く必要があります（手動同期）。 これが煩わしい場合は、マーカーだけ書いて拡張専用ドキュメントにする選択も可能。
