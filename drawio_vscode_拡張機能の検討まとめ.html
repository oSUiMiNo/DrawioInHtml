HTML埋め込みDrawioをVSCodeで編集する — 検討まとめ

結論 実現は可能。さらに「既存の仕組みを使うショートカット経路」が複数存在する。「既存HTML表示の仕組み」が何を指すかで難易度が大きく変わるため、まずそこを切り分けるのが最短ルート。

① 最大のショートカット：「.drawio.svg ファイル」を活用推奨

知識カットオフ後（2026年5月時点）の最新版でも、hediet.vscode-drawio（v1.9.0系）は健在で、これが鍵。

仕組み

  • *.drawio.svg というファイルは「正当なSVG画像」かつ「Drawio編集データ」の二重構造（SVG内部にXMLが埋め込まれている）。
  • HTML側は <img src="diagram.drawio.svg"> で普通に表示するだけ。
  • そのファイルをVSCodeで開くと hediet.vscode-drawio が起動して Drawio UI で編集できる。保存するとSVG表示部分も自動で更新される。
  • HTMLビュアー（例：Live Preview拡張）は何も知らないまま、最新の図を再表示する。
メリット 自作実装ゼロで「HTML埋め込みDrawioをその場で編集」が成立する。.drawio.png でも同じ仕組みが使える。

② 「HTML本文の中に直接Drawio XMLを書き込む」場合

ここからは話が変わる。編集機能を“どこに”寄生させるかで3つの候補に分かれる。

候補A：VSCode組み込みの Markdown Preview を拡張

  • 公式が markdown.markdownItPlugins という拡張ポイントを公開している。
  • Markdown Preview Mermaid Support が Mermaid でやっている手法と同じ流れ。
  • Drawio公式の Embed Mode（https://embed.diagrams.net/?embed=1&proto=json）を <iframe> で差し込み、postMessage で load / save を仲介するブリッジを書くだけ。
  • 既存類似品 dzylikecode.vscode-drawio-plugin-markdown がほぼこの構成 → フォークするのが最速。

候補B：HTMLファイル自体のプレビューを拡張

  • Microsoft純正の Live Preview 拡張はプラグイン拡張ポイントを公開していない。
  • そのため、既存の仕組みに乗ることができず、結局自作Custom Editor / Webview を一から作ることになる。
  • 「ショートカット」要件には合わない。

候補C：Drawio公式 Embed Mode を WebView で包んだ薄い自作拡張

  • 30〜50行レベルでプロトタイプ可能。
  • Drawioエディタ機能はすべて本家iframeに丸投げ。
  • 自分が書くのは「保存ボタン押下 → XML受け取り → HTMLに書き戻す」だけ。

③ 判断フロー

HTMLに「画像参照」で十分? ├─ Yes → ★ .drawio.svg + hediet拡張で完了（実装ゼロ） └─ No（HTML内にXMLを直接埋めたい） │ ソースがMarkdown? ├─ Yes → 候補A：Markdown Preview拡張（既存をフォークが最速） └─ No（純HTML） → 候補C：Embed ModeをWebViewで包む薄い自作拡張

④ 「機能が無限」問題への根本回答

原則 Drawioエディタを自分で再実装しないこと。これが唯一の正解。

embed=1&proto=json パラメータ付きで本家エディタを <iframe> として読み込めば、シェイプ追加・スタイル・エクスポートなどの新機能は本家アップデートで自動的についてくる。あなたが書くべきは「保存イベントを受け取って書き戻す」入出力ブリッジのみ。

⑤ 比較表

経路 実装量 編集できる対象 推奨度
① .drawio.svg + hediet拡張 ゼロ HTMLから画像参照されたDrawio図 ★★★
候補A：Markdown Preview拡張 小（既存フォーク） Markdown内に埋め込まれたDrawio ★★

⑥ 推奨アクション

  1. まず「.drawio.svg + hediet拡張」で要件が満たせるかを試す。これで足りるなら拡張機能の自作は不要。
  2. 不足する場合のみ、ソースがMarkdownか純HTMLかで候補A／Cに分岐する。
  3. いずれの場合も「Drawioエディタ本体を自作しない」のが鉄則。本家 Embed Mode を使い倒す。
